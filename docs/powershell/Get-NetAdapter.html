Get-NetAdapter

The Get-NetAdapter command exposes MSFT_NetAdapter instances with 121 properties. Some of them have no values for any adapters, some properties expose the same information as other properties in different ways. This is an attempt to find some rhyme and reason to the available information.

Here's a command that should display most of what you could be looking for, feel free to remove anything you would not be interested in.

Get-NetAdapter | Select-Object Name,DriverDescription,DriverProvider,ComponentID,PnPDeviceID,`
DriverName,InterfaceName,MediaType,DriverDate,DriverVersion,NdisVersion,DeviceID,InterfaceIndex,`
HigherLayerInterfaceIndices,LowerLayerInterfaceIndices,MacAddress,PermanentAddress,MtuSize,LinkSpeed,`
AdminStatus,ifOperStatus,MediaConnectionState,Status,SystemName,AdminLocked,Hidden,IMFilter,`
iSCSIInterface,PromiscuousMode,NotUserRemovable,ConnectorPresent,Virtual,WdmInterface,`
DeviceWakeUpEnable,OperationalStatusDownDefaultPortNotAuthenticated,`
OperationalStatusDownInterfacePaused,OperationalStatusDownLowPowerState,`
OperationalStatusDownMediaDisconnected,EnabledDefault,State,RequestedState,TransitioningToState,`
PortNumber,EnabledState,NdisMedium,NdisPhysicalMedium,InterfaceAdminStatus,InterfaceType,NetLuid,`
NetLuidIndex,EndPointInterface,HardwareInterface,FullDuplex,CreationClassName,`
SystemCreationClassName | Out-GridView

String information

The Name property contains a string identifier.

The InterfaceAlias and ifAlias properties contains the same information.

Example value: VirtualBox Host-Only Network

The DriverDescription property contains additional useful information about the driver.

The InterfaceDescription and ifDesc properties contains the same information.

Possible values include:

  • Cisco AnyConnect Secure Mobility Client Virtual Miniport Adapter for Windows x64
  • Hyper-V Virtual Ethernet Adapter
  • Intel(R) Wi-Fi 6 AX200 160MHz
  • TAP-Windows Adapter V9
  • VirtualBox Host-Only Ethernet Adapter
  • Wintun Userspace Tunnel

The DriverProvider property contains the company name that developed the driver. Possible values include:

  • Cisco Systems
  • Intel
  • Microsoft
  • Oracle Corporation
  • TAP-Windows Provider V9
  • WireGuard LLC

The Hardware ID, aka PnP component ID, is available in ComponentID. Possible values include:

  • PCI\VEN_8086&DEV_2723&SUBSYS_40808086
  • root\tap0901
  • sun_vboxnetadp
  • vms_mp
  • VPNVA
  • Wintun

The PnPDeviceID holds somewhat similar but different information

  • PCI\VEN_8086&DEV_2723&SUBSYS_40808086&REV_1A\4&1d510a7f&0&00E0
  • ROOT\NET\0000
  • ROOT\NET\0001
  • ROOT\NET\0002
  • ROOT\NET\0003
  • ROOT\VMS_MP\0000

The DriverFileName property contains the name of the driver file. Possible values include:

  • Netwtw10.sys
  • tap0901.sys
  • VBoxNetAdp6.sys
  • VmsProxyHNic.sys
  • vpnva64-6.sys
  • wintun.sys

The DriverName property contains the path to the driver file.

Example value: \SystemRoot\system32\DRIVERS\VBoxNetAdp6.sys

The InterfaceName property contains a string prefix and a number suffix.

The ifName property contains the same information.

Example value: ethernet_32774

Possible prefixes:

  • ethernet
  • iftype53
  • wireless

The MediaType property contains a string with the following possible values on my system:

  • IP
  • Native 802.11
  • 802.3

The PhysicalMediaType property is the same for some adapters, but Unspecified for others:

  • Unspecified
  • Native 802.11
  • 802.3

Date and version information

The DriverInformation property contains the driver date, version and NDIS in a string.

Example value: Driver Date 2022-07-19 Version 6.1.36.2435 NDIS 6.0

The DriverDate property exposes a date as a string

Example value: 2022-07-19

The DriverDateData property exposes DriverDate as a 64-bit unsigned integer.

Example value: 133026624000000000

The driver's version is available in DriverVersion and DriverVersionString as a string.

Example value: 6.1.36.2435.

The driver's NDIS version is available in DriverMajorNdisVersion and DriverMinorNdisVersion as bytes.

Example values: 6 and 30.

The full NDIS version is available in NdisVersion.

Example value: 6.30

The driver's version is available in MajorDriverVersion and MinorDriverVersion as 16-bit unsigned integers.

Example values: 9 and 24.

GUIDs

The DeviceID property exposes a GUID as a string with braces.

The InterfaceGuid and InstanceID properties exposes the same information.

Example value: {545C58EC-A1C2-468B-ADD7-00E3A9604DA2}

The DeviceName property exposes the same GUID prefixed with \Device\

Example value: \Device\{545C58EC-A1C2-468B-ADD7-00E3A9604DA2}

Interface indices

The InterfaceIndex property exposes a 32-bit unsigned integers.

The ifIndex property contains the same information.

The HigherLayerInterfaceIndices and LowerLayerInterfaceIndices properties are arrays of 32-bit unsigned integers.

On my system, HigherLayerInterfaceIndices had exactly one value for each adapter.

On my system, LowerLayerInterfaceIndices had one value for HyperV, nothing for the other adapters.

MAC address

MAC addresses are available in MacAddress and LinkLayerAddress as strings.

Example value: 00-B0-D0-63-C2-26.

The same information is available in the NetworkAddresses property as a string too.

Example value: {00B0D063C226}.

The same information is available in the PermanentAddress property as a string too, but was different from the MAC address for HyperV.

Example value: 00B0D063C226.

MTU

The Maximum Transmission Unit can be found in MtuSize as an unsigned 32-bit integer and in ActiveMaximumTransmissionUnit as an unsigned 64-bit integer

Example values: 0, 1500, 6535.

Speed

The Speed, ReceiveLinkSpeed and TransmitLinkSpeed propertiess exposes the interface speed as a 64-bit unsigned integer.

Example values in increasing order: 0, 866700000, 1000000000, 10000000000, 100000000000

The LinkSpeed property exposes the interface speed as a formatted string.

Example values in increasing order: 0 bps, 866.7 Mbps, 1 Gbps, 10 Gbps, 100 Gbps

Up or down

A number of properties are related to whether the adapter is up, down, or something in-between.

The AdminStatus property exposes an enum value of type NET_IF_ADMIN_STATUS. Most adapters were Up according to this property.

Values in Microsoft.PowerShell.Cmdletization.GeneratedTypes.NetAdapter.NET_IF_ADMIN_STATUS include:

  • Down
  • Testing
  • Up

The ifOperStatus property exposes an enum value of type IF_OPER_STATUS. VPN adapters were Down according to this property,other adapters were Up, one adapter had NotPresent.

Values in Microsoft.PowerShell.Cmdletization.GeneratedTypes.NetAdapter.IF_OPER_STATUS include:

  • Dormant
  • Down
  • LowerLayerDown
  • NotPresent
  • Testing
  • Unknown
  • Up

The InterfaceOperationalStatus property seems to expose ifOperStatus as an unsigned 32-bit integer with the corresponding enum's number.

The MediaConnectionState property exposes an enum value of type NET_IF_MEDIA_CONNECT_STATE. VPNs appeared Disconnected, wifi, virtual box and HyperV Connected, and another appeared as Unknown.

Values in Microsoft.PowerShell.Cmdletization.GeneratedTypes.NetAdapter.NET_IF_MEDIA_CONNECT_STATE include:

  • Connected
  • Disconnected
  • Unknown

The MediaConnectState property seems to expose MediaConnectionState as an unsigned 32-bit integer with the corresponding enum's number.

The Status property seems to expose MediaConnectionState as a string, but with different values:

  • Up
  • Disconnected
  • Not Present

Other properties

The SystemName property is the computer name.

The AdminLocked property was always false on my system.

The Hidden property was always false on my system.

The IMFilter property was always false on my system.

The iSCSIInterface property was always false on my system.

The PromiscuousMode property was always false on my system.

The NotUserRemovable property was true only for HyperV.

The ConnectorPresent property was true only for wifi.

The Virtual property was false only for wifi.

The WdmInterface property was true for some but not others.

The DeviceWakeUpEnable property was true only for wifi.

The OperationalStatusDownDefaultPortNotAuthenticated property was always false.

The OperationalStatusDownInterfacePaused property was always false.

The OperationalStatusDownLowPowerState property was always false.

The OperationalStatusDownMediaDisconnected property was true for some adapters.

The EnabledDefault property was always 2 as an unsigned 16-bit integer.

The State property was always 2 as an unsigned 32-bit integer.

The RequestedState property was always 12 as an unsigned 16-bit integer.

The TransitioningToState property was always 12 as an unsigned 16-bit integer.

The PortNumber property was always 0 as an unsigned 16-bit integer.

The EnabledState property was always 5 as an unsigned 16-bit integer.

The NdisMedium property is a 32-bit unsigned integer, which was mostly 0, but 16 for on adapter, and 19 for another.

The NdisPhysicalMedium property is a 32-bit unsigned integer, which was mostly 0, but 9 for on adapter, and 14 for another, and not the same adapters as above except wifi which had values for both properties..

The InterfaceAdminStatus is exposed as an unsigned 32-bit integer. It was 1 for most adapters, 2 for one adapter.

The InterfaceType is exposed as an unsigned 32-bit integer.

Example values: 6, 53, 71.

The NetLuid property is an unsigned 64-bit integer which hold large values.

Example value: 14918723538255872

The NetLuidIndex property is an unsigned 32-bit integer which hold values typically around 32768.

Example values: 32768, 32769, 32774, 32777, 32778

The EndpointInterface property was true only for the VirtualBox adapter.

The HardwareInterface property was true only for wifi.

The FullDuplex property was true for the VirtualBox and OpenVPN adapters, false for wifi, empty for others .

The MediaDuplextState seems to expose the same information as a 32-bit unsigned integer, with 0 for nothing, 1 for False and 2 for True .

The CreationClassName property represents the underlying object class presented by the CIM object, because Get-NetAdapter uses CIM session to retrieve the information. In this case, it is MSFT_NetAdapter.

The SystemCreationClassName was always CIM_NetworkPort.
